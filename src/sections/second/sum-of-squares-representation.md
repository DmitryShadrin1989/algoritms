# Представить натуральное число как сумму квадратов

## Условие

Дано натуральное число `n`. Вернуть **список целых** чисел `a₁, a₂, …`, таких что:

\[
a_1^2 + a_2^2 + \cdots = n
\]

**Дополнительно (если базовая часть решена):** список должен иметь **минимально возможную длину** (минимальное число слагаемых-квадратов).

### Примеры (как в шаблонах)

| `n` | Один из допустимых списков |
|-----|----------------------------|
| `5` | `[2, 1]` — так как `2² + 1² = 5` |
| `9` | `[3]` — так как `3² = 9` |
| `11` | `[3, 1, 1]` — так как `3² + 1² + 1² = 11` |

Для **минимальной длины** у `11` это как раз **3** слагаемых (двумя квадратами `11` не представить).

## Связь с LeetCode

[Perfect Squares](https://leetcode.com/problems/perfect-squares/) — там обычно просят только **минимальное количество** квадратов (число), а не сам список оснований. Здесь формулировка шире: сначала любое разложение, затем — **оптимальная длина** и при желании восстановление списка.

## Ожидания на интервью

- **Подойдёт** переборный вариант (например, наращивание длины списка, backtracking, BFS по состоянию «остаток») с ростом сложности при большом `n`.
- **Плюс**, если кандидат знает **теорему Лагранжа о четырёх квадратах**: любое натуральное число есть сумма **не более четырёх** квадратов — значит **минимальная** длина ответа **не больше 4** (это сильно сужает поиск для оптимальной версии).
- **Плюс** за **«вырожденное»** решение: `n` раз число `1` (`1² + … + 1² = n`) — валидно для первой части задачи без минимизации длины.
- **Плюс** за **жадный** алгоритм («всегда вычитать наибольший возможный полный квадрат») и **контрпример**, где он даёт **не минимальное** число слагаемых. Например, `12 = 9 + 1 + 1 + 1` (жадно по одному большому квадрату — **4** слагаемых), тогда как оптимально `12 = 4 + 4 + 4` (**три** квадрата по `2`).
- **Плюс** за **оптимальное** решение: динамика вида размена монет (`dp[r] = min` по `dp[r - k²] + 1`) для **минимального счёта**, с восстановлением списка по предшественникам; или BFS по остатку с первым достигнутым `0` на минимальной глубине.

## Кратко про сложность (оптимальная длина)

- DP по остатку до `n`: время **`O(n · √n)`**, память **`O(n)`** для массива `dp`.
- С учётом теоремы Лагранжа можно добавить эвристики (проверка 1, 2, 3, 4 слагаемых) для ускорения в среднем.
