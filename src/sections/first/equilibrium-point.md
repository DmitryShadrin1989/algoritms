# Точка равновесия (equilibrium point) массива

## Условие

Дан массив целых чисел. Нужно найти **индекс** `i`, такой что **сумма элементов слева** от `i` **равна** **сумме элементов справа** от `i` (элемент с индексом `i` **не входит** ни в левую, ни в правую сумму).

Числа могут быть **отрицательными**. Точки равновесия **может не быть**.

**Сложность:** время **O(n)**, дополнительная память желательно **O(1)**.

## Модификация для интервью

Ту же идею можно сформулировать для **односвязного / двусвязного списка**, **стека** (если разрешён доступ ко всем элементам за линейное время) и т.п.: сначала найти **сумму всех элементов** одним проходом, затем вторым проходом поддерживать **накопленную левую сумму** и проверять условие равновесия на каждом узле / позиции.

## Хорошее решение: O(n) времени, O(1) памяти

1. Один проход: **полная сумма** `total` массива.
2. Второй проход: `leftSum = 0`. Для каждого индекса `i`:
   - **правая сумма** (без `array[i]`): `rightSum = total - leftSum - array[i]`;
   - если `leftSum == rightSum`, вернуть `i`;
   - `leftSum += array[i]`.

Эквивалентная проверка без явной `rightSum`: `leftSum == total - leftSum - array[i]` ⟺ `2 * leftSum + array[i] == total` (аккуратно с переполнением — при необходимости считать в `long`).

## Удовлетворительно

Дополнительный массив **префиксных** сумм (или два массива — префиксы и суффиксы), затем линейный поиск индекса. **O(n)** памяти.

## Плохо

Вложенные циклы: на каждом `i` заново считать сумму справа (или слева). **O(n²)**.

## Краевые случаи

- **Пустой массив** — решения нет (например, вернуть `-1` или бросить исключение — зафиксировать в контракте).
- **Один элемент** — слева `0`, справа `0` → индекс `0` обычно является точкой равновесия.
- **Два и более элемента** — проверить вручную на маленьких примерах.
- **Нет равновесия** — вернуть согласованное с API значение (часто `-1` или `OptionalInt.empty()`).

При больших по модулю значениях имеет смысл считать суммы в **`long`**, чтобы не словить переполнение `int`.
