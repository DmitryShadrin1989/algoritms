# Посчитать количество левых листьев бинарного дерева

## Условие

**Левый узел** — узел, который является **левым ребёнком** своего родителя.

**Лист** — узел **без** детей (`left == null` и `right == null`).

Нужно посчитать **количество** узлов, которые одновременно **левые** и **листья**.

## Расширение

В узлах хранятся **числовые значения**. Посчитать **сумму** значений во всех **левых листьях** (логика «левизны» и «листовости» та же).

## Подход

Подойдёт **любой обход** дерева с проверкой нужных свойств. Чаще всего удобен **рекурсивный DFS**.

### Почему «не только по самому узлу»

По одному узлу **нельзя** понять, левый он у родителя или правый: это свойство **ребра** родитель → ребёнок. Значит:

- либо при рекурсии **передавать флаг** `isLeft` (корень вызывают с `isLeft = false`);
- либо **заглядывать вперёд**: лист — это узел, у которого **левый** ребёнок существует, является листом, и мы считаем именно этого ребёнка как «левый лист» (вариант без флага, но смотрим `node.left` и его детей).

Проще и нагляднее — **флаг в параметрах** рекурсии.

### Краевые случаи (ожидания от кандидата)

- **Дерево из одного узла (корень):** корень **не** является ни левым, ни правым ребёнком (у него нет родителя). Обычно это **не** левый лист → ответ **0** (или кандидат **сам уточняет** определение у интервьюера — это плюс).
- **`null` на входе:** либо явно **запрещён** контрактом, либо возвращаем **0** (и обрабатываем в начале функции). Хорошо, если кандидат это **проговаривает**.

## Идея рекурсии (счётчик)

```text
dfs(node, isLeft):
  if node == null: return 0
  if node.left == null && node.right == null && isLeft: return 1
  return dfs(node.left, true) + dfs(node.right, false)
```

Вызов с корня: `dfs(root, false)`.

## Идея рекурсии (сумма значений)

Тот же флаг `isLeft`; если узел — левый лист, добавляем `node.val`, иначе суммируем результаты по детям.

## Источник

LeetCode (сумма левых листьев): [Sum of Left Leaves](https://leetcode.com/problems/sum-of-left-leaves/)

Для постановки «только количество» достаточно заменить сумму на инкремент счётчика.
