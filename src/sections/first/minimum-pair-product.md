## Критерии оценки

**[30 мин, макс. 4 балла]**

### Алгоритм (с учётом отрицательных чисел)

- **1 балл** — корректно за `O(n²)` или хуже;
- **2 балла** — корректно за `O(n log n)` (например, `Collections.sort`);
- **3 балла** — за **линейное** время `O(n)` (допускается подсказка, что так можно).

### Corner case’ы

*(В качестве подсказки можно спросить кандидата, какие крайние случаи бывают.)*

- **+0.25** — проверка, что размер входа **≥ 2**;
- **+0.25** — учтены **отрицательные** значения;
- **+0.25** — учтено **переполнение** при перемножении (например, считать в `long` / `BigInteger`);
- **+0.25** — корректно найдены два минимальных и/или два максимальных значения (не перепутаны сравнения с `Integer.MAX_VALUE` / `MIN_VALUE`).

## Тест-кейсы

| Вход | Ожидаемый результат |
|------|---------------------|
| `{1, 2, 3, 4}` | `2` |
| `{-1, -2, -3, -4}` | `2` |
| `{-1, -2, 3, 4}` | `-8` |
| `{0, 0, 0, 0}` | `0` |

## Идея линейного решения

За один проход находим два **наименьших** (`minVal1 ≤ minVal2`) и два **наибольших** (`maxVal1 ≥ maxVal2`) элемента.

Минимум по всем парам достигается одним из вариантов:

- все числа **≥ 0** → минимальное произведение у двух наименьших: `minVal1 * minVal2`;
- все числа **< 0** (строго отрицательные) → минимальное произведение у двух **наибольших** (ближе к нулю): `maxVal1 * maxVal2` (произведение двух отрицательных положительно, и оно минимально среди таких пар);
- **смешанный** знак (или нули «ломают» чистую картину) → кандидат `minVal1 * maxVal1` (самый маленький × самый большой даёт сильно отрицательное произведение).

На практике эквивалентно взять `min(minVal1 * minVal2, maxVal1 * maxVal2, minVal1 * maxVal1)` при аккуратном использовании `long`.

## Ожидаемый код (Java)

```java
public static long findMinProduct(int[] arr) {
    if (arr.length < 2) {
        throw new RuntimeException();
    }
    // [minVal1, minVal2, ..., maxVal2, maxVal1]
    long minVal1 = Integer.MAX_VALUE;
    long minVal2 = Integer.MAX_VALUE;
    long maxVal1 = Integer.MIN_VALUE;
    long maxVal2 = Integer.MIN_VALUE;
    for (int curVal : arr) {
        if (curVal < minVal1) {
            minVal2 = minVal1;
            minVal1 = curVal;
        } else if (curVal < minVal2) {
            minVal2 = curVal;
        }

        if (curVal > maxVal1) {
            maxVal2 = maxVal1;
            maxVal1 = curVal;
        } else if (curVal > maxVal2) {
            maxVal2 = curVal;
        }
    }

    if (minVal1 >= 0 && minVal2 >= 0) {
        return minVal1 * minVal2;
    } else if (maxVal1 < 0 && maxVal2 < 0) {
        return maxVal1 * maxVal2;
    } else {
        return minVal1 * maxVal1;
    }
}
```
