# Объединение (слияние) или пересечение сортированных списков

Есть два сортированных списка (массива). Нужно написать функцию, которая строит **новый** сортированный список — либо **объединение**, либо **пересечение** элементов этих двух списков (два варианта задачи: можно дать любой или оба).

## Пример

**1-й список:** 1, 2, 2, 5, 7, 14  
**2-й список:** 4, 6, 6, 7, 9, 14, 15

- **Объединение как при слиянии двух отсортированных списков (merge):** 1, 2, 2, 4, 5, 6, 6, 7, 7, 9, 14, 14, 15 — в результат попадают **все** элементы обоих массивов с исходными кратностями, порядок не убывает (аналог `std::merge`).
- **Пересечение по правилу `min(n, m)`:** 7, 14

> **Замечание:** если для «объединения» требовать правило **`max(n, m)`** копий на значение, результат для этих входных данных **другой** (например, для `7` и `14` было бы по одной копии, а не по две). Имеет смысл явно выбрать в формулировке либо **merge**, либо **multiset union по max**.

## Семантика дубликатов

Важно **заранее** договориться о поведении при повторах. Естественный вариант для мультимножеств:

- если элемент `x` в первом списке встречается `n` раз, во втором — `m` раз, то в **пересечении** оставлять **`min(n, m)`** копий;
- в **объединении как multiset union** — **`max(n, m)`** копий (это **не** то же самое, что построчное слияние двух списков **merge**).

Можно ослабить условие: например, заранее сказать, что внутри одного списка дубликатов нет, или потребовать другую семантику (уникальные значения, только из первого списка и т.д.).

**Инвариант формулировки:** обычно просят **вернуть новый** список, а не менять входные массивы на месте — элементы нужно **скопировать** в результат.

## Ориентиры по реализации

Имеет смысл сравнить с поведением стандартных алгоритмов (для отсортированных диапазонов):

- `std::set_intersection` — пересечение с учётом кратностей (`min`);
- `std::set_union` — объединение в смысле **`max(n, m)`** на каждом значении;
- `std::merge` — все элементы обоих входов подряд в порядке неубывания (как в примере «Объединение» выше).

См. также документацию к этим алгоритмам и заранее зафиксируйте, какой смысл «объединения» нужен в интервью.
