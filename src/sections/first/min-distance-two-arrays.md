# Минимальная разность элементов двух массивов

## Условие

Даны два массива `a` и `b`. Нужно найти минимум по всем парам индексов:

\[
\min_{i,\,j} \lvert a[i] - b[j] \rvert
\]

### Сигнатура (C++)

```cpp
int minDistance(std::vector<int>& a, std::vector<int>& b);
```

### Примеры (assert)

```cpp
assert(minDistance({0}, {1}) == 1);
assert(minDistance({10, 0, 2}, {5, 100, 12}) == 2);
```

**Разрешено:** модифицировать массивы **на месте** (например, отсортировать).

**Требование к памяти:** без **существенной** дополнительной памяти — имеется в виду решение за **линейный проход** после сортировки, без вспомогательных структур размера `O(n)` (сама **in-place** сортировка вроде `std::sort` может использовать `O(\log n)` стека или небольшой буфер — это обычно оговаривают отдельно).

## Хорошее решение

1. Отсортировать **каждый** массив по возрастанию.
2. **Два указателя** `i` по `a`, `j` по `b`:
   - обновить ответ: `min(|a[i] - b[j]|)`;
   - если `a[i] < b[j]`, сдвинуть `i++`, иначе `j++` (двигать указатель на **меньший** текущий элемент, чтобы уменьшить разрыв).

**Время:** `O(n \log n + m \log m)` на сортировку + `O(n + m)` на merge-подобный проход.

```cpp
std::sort(a.begin(), a.end());
std::sort(b.begin(), b.end());
long long best = LLONG_MAX;  // или INT_MAX по контракту для непустых входов
size_t i = 0, j = 0;
while (i < a.size() && j < b.size()) {
    best = std::min(best, std::llabs(static_cast<long long>(a[i]) - b[j]));
    if (a[i] < b[j]) {
        ++i;
    } else {
        ++j;
    }
}
```

Для **отсортированных** массивов цикла `while (i < a.size() && j < b.size())` с правилом «сдвигаем указатель на меньший элемент» достаточно: к моменту выхода одного указателя за границу минимум `|a[i]-b[j]|` уже учтён.

## Слабее: `lower_bound` / `upper_bound`

Допустимо как запасной вариант: для каждого элемента одного массива бинарный поиск во втором. Кандидат должен **чётко понимать**, что возвращают `lower_bound` и `upper_bound`, и без ошибок обработать соседей итератора.

**Минус:** если в условии заменить на **уже отсортированные** массивы, кандидат с таким подходом может **не выйти на линейный** двухуказательный проход — для интервью это **серьёзный минус** по сравнению с merge-стилем.

## Краевые случаи (ожидания от кандидата)

- **Пустые массивы** — заранее не оговорено; разумно **спросить**. Вариант контракта: вернуть `INT_MAX` (или бросить исключение — зафиксировать в условии).
- **Переполнение:** например, `minDistance({INT_MAX}, {-1})` — модуль разности **не помещается** в `int`. Сильный плюс, если кандидат это замечает и считает разницу в **`long long`** (или иначе избегает UB).

## Идея проверки ответа в коде

```cpp
long long diff = std::llabs(static_cast<long long>(a[i]) - static_cast<long long>(b[j]));
```

Итог можно приводить к `int`, только если по условию гарантировано, что минимальная разность помещается в `int`.
