# Наибольшая подстрока одинаковых букв

**Задача не рекомендуется к использованию** (например, слишком узкая / часто «зубрёжка»).

## Условие

Задана строка `S` из **строчных латинских** букв. Нужно найти **длину** наибольшей подстроки, в которой **все буквы одинаковы**.

### Примеры

| Вход | Ответ |
|------|------|
| `""` | `0` |
| `"a"` | `1` |
| `"abbc"` | `2` |
| `"adddaabaa"` | `3` |

## Плохое решение

Квадратичный перебор: фиксируем стартовую позицию и тянем подстроку вправо, пока символы совпадают — **O(n²)**.

## Хорошее решение: O(n), два указателя

Один проход: поддерживаем начало текущего блока одинаковых символов `i` и «бегунок» `j`. Пока `s[j] == s[i]`, двигаем `j`. Длина блока `j - i`, затем перескакиваем `i` на `j`.

```cpp
int getLen(const std::string& s) {
    int ans = 0;
    const int n = static_cast<int>(s.length());
    for (int i = 0, j = 0; i < n; i = j) {
        while (j < n && s[i] == s[j]) {
            ++j;
        }
        ans = std::max(ans, j - i);
    }
    return ans;
}
```

## Усложнение 1: не более **двух** различных символов в подстроке

Нужна **наибольшая длина** подстроки, в которой встречается **не более 2** различных букв.

### Примеры

| Вход | Ответ |
|------|------|
| `"ab"` | `2` |
| `"aaa"` | `3` |
| `"abaccab"` | `4` |
| `"baacabb"` | `4` |

**Подход:** скользящее окно `[left, right]` с подсчётом частот символов в окне; пока различных символов ≤ 2, расширяем `right`; как только стало 3 — сдвигаем `left`, пока снова не останется ≤ 2. Поддерживаем максимум длины окна. При алфавите из 26 букв счётчик — массив на 26, **O(n)** по времени, **O(1)** по памяти (размер алфавита константа).

---

## Усложнение 2: не более **K** различных символов

Та же схема **скользящего окна**: расширяем правый указатель, поддерживаем число **различных** символов в окне; если оно превысило `K`, двигаем левый указатель и уменьшаем счётчики, пока условие не восстановится. **Время O(n)** (каждый индекс входит и выходит из окна не более одного раза), память **O(A)** на счётчики, `A` — размер алфавита (для латиницы `A ≤ 26` — фактически **O(1)**).

**Замечание:** идея «хранить индекс последнего вхождения каждой буквы, брать минимум и двигать левый границу» ближе к задаче **без повторяющихся символов** (или к вариантам с «последним конфликтующим» индексом). Для **«не более K различных»** на интервью обычно ожидают именно **два указателя + частоты** (при необходимости с ordered map для подсчёта различных — тогда возможна оценка порядка **O(n log A)**).

---

## Кратко про сложность (как в исходных заметках)

- Базовая задача (все буквы одинаковы): **O(n)**.
- Не более 2 различных: **O(n)** при массиве счётчиков на алфавит.
- Не более K различных: **O(n)** при фиксированном алфавите и массиве счётчиков; при структурах с логарифмом на операцию — возможны варианты порядка **O(n log A)**.
