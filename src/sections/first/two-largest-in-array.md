# Вернуть два наибольших числа в массиве

## Условие

Дан массив; нужно вернуть **два наибольших** значения (в каком порядке — уточнить: обычно сначала максимум, потом второй; при дубликатах максимума вторым может быть то же значение, если в массиве два одинаковых максимума).

Задачу можно ставить для **`int`**, а **`float` / `double`** интереснее: **строгое сравнение на равенство** с плавающей точкой часто бессмысленно; имеет смысл обсуждать **epsilon**, порядок сравнений или формулировку «два наибольших по порядку» без привязки к `==`.

## Модификация: максимальное произведение **пары** элементов

Нужно не два максимума как таковых, а пара с **максимальным произведением**. Тогда кандидатам нужны **два наибольших** и **два наименьших** (произведение двух больших отрицательных может победить).

## Ход интервью

- Если предлагают **сортировку** — остановить и попросить **линейный** алгоритм `O(n)` и `O(1)` доп. памяти.
- После линейного решения можно развить тему: **M наибольших** из `N` элементов — сортировка, **куча** размера `M`, **quickselect** и т.д.

## Типичные ошибки

На простой задаче часто всплывают такие баги:

1. **Нет обработки** массива длины **0** или **1** (или некорректный контракт / исключение).
2. **Неверная инициализация** первого и второго максимума (например, `0`, когда в массиве все числа отрицательные).
3. После корректной инициализации цикл начинается с **индекса 0**, и **второй максимум перезаписывается** тем же элементом, что уже стал `max`.
4. **Потеря второго результата** — нет ветки `else if` для обновления только `second`:

```text
if (array[i] > max) {
    second = max;
    max = array[i];
}
// Нет else if (array[i] > second) — значения между second и max не учитываются.
```

5. **Потеря текущего максимума** — при `array[i] > max` обновляют только `max`, не перенося старый `max` в `second`:

```text
if (array[i] > max)
    max = array[i];   // старый max потерян
else if (array[i] > second)
    second = array[i];
```

## Рабочий однопроходный паттерн (идея)

```text
// Предположим, length >= 2; иначе — отдельный контракт.
if (array[0] >= array[1]) {
    max = array[0]; second = array[1];
} else {
    max = array[1]; second = array[0];
}
for i from 2 to n-1:
    x = array[i]
    if (x > max) {
        second = max
        max = x
    } else if (x > second) {
        second = x
    }
```

Для **дубликатов максимума** (например `[5, 5, 3]`): при втором `5` сработает `else if (x > second)` и даст `second == 5`, если нужны **два** наибольших **значения** с учётом повторов. Если нужны **два различных** индекса / два разных значения — это **отдельная** постановка.

Для **`float`**: те же сравнения `>` / `>=`; избегать проверок `a == b` для ветвления логики «равны ли два максимума», если это не оговорено явно.

## Сложность

- Один проход: **O(n)** время, **O(1)** память (плюс возвращаемая пара).
