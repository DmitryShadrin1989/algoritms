# Построить бинарное дерево поиска из отсортированного массива

## English

Given an array where elements are sorted in ascending order, convert it to a **height-balanced** binary search tree.

## Условие

Дан массив целых чисел, упорядоченный по **возрастанию**. Нужно построить по нему **сбалансированное по высоте** бинарное дерево поиска (BST): для каждого узла высоты левого и правого поддеревьев отличаются не более чем на 1.

### Пример (один из возможных ответов)

Для `[-10, -3, 0, 5, 9]` возможно дерево:

```text
        0
     /     \
   -3       5
  /  \    /  \
-10   x  x    9
```

(`x` — отсутствующие дочерние узлы.)

## Алгоритм, сложность, краевые случаи

**Идея:** в отсортированном массиве **медиана** (середина отрезка `[left, right]`) — естественный кандидат в корень: в левом подмассиве все элементы меньше, в правом — больше, глубины поддеревьев примерно равны.

Рекурсивно:

1. Если `left > right` — вернуть `null` (пустой отрезок).
2. `mid = left + (right - left) / 2` — так безопаснее, чем `(left + right) / 2`, при очень больших индексах (нет переполнения `int`). Для чётной длины отрезка можно брать левый из двух «центральных» или договориться о другом правиле — главное, чтобы дерево оставалось BST и сбалансированным по высоте.
3. Корень = `nums[mid]`, `left` = рекурсия на `[left, mid - 1]`, `right` = рекурсия на `[mid + 1, right]`.

**Время:** `O(n)` — каждый элемент массива один раз становится узлом.

**Память:** `O(h)` дополнительной стека рекурсии, `h = O(log n)` при сбалансированном построении; плюс `O(n)` на само дерево.

**Краевые случаи:**

- пустой массив → `null`;
- один элемент → лист без детей;
- два элемента — уточнить выбор `mid` (типично корень = левый элемент, правый — правое поддерево), баланс высоты сохраняется.
